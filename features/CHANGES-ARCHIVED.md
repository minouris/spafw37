## Column Definitions

- **Change ID**: Stable identifier in format `{subject}-{verb}-{NNN}` (e.g., `feat-add-001`, `tool-fix-003`, `docs-add-001`)
  - Subjects: `feat` (feature), `tool` (tools), `docs` (documentation)
  - Verbs: `add` (add new), `enh` (enhance), `ref` (refactor), `fix` (bugfix)
  - Numbers: 3-digit sequential per subject (001-999)
- **Component**: Reference to component from CHANGES-ACTIVE.md identification registry (3-5 letters per hyphenated group for clarity)
- **Ticket ID**: GitHub issue number
- **Description**: Short description of the change
- **Opened**: Date the change was created (YYYY-MM-DD format)
- **Closed**: Date the change was closed (YYYY-MM-DD format)
- **Status**: Final state (typically "Complete" or "Rejected")

**Milestone Sections:** Changes are organized by milestone (e.g., "v1.1.0", "No Milestone"). The milestone is implicit from the section heading. The most recent milestone appears first, followed by "No Milestone" (for infrastructure/tooling work), then older milestones in reverse chronological order.

**Note:** Rejected items are displayed with strikethrough formatting across all columns except Status.

## About This Registry

This file contains all closed and completed changes to the spafw37 project. Active changes are tracked in [CHANGES-ACTIVE.md](CHANGES-ACTIVE.md).

**Purpose:**
- Historical record of all completed and rejected changes
- Audit trail for project evolution
- Reference for similar future work
- Understanding of rejected proposals and reasons
- Change velocity and metrics data

**Milestone Organization:** Changes are organized with the most recent milestone first, followed by "No Milestone" (for infrastructure/tooling work not tied to product releases), then older milestones in reverse chronological order.
